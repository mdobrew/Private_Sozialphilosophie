## 2. Programme Parameters

### 2.1 Capacity envelope (the binding constraint) + motivational target

The programme is defined by the **capacity available to the reader**, not by a total-hours commitment. Timeline is a *derived* output of capacity × scope. The binding parameters are the capacity numbers below.

**Motivational target — soft anchor, not rigid commitment.** A ~4-year target (give-or-take) is psychologically load-bearing: an open-ended self-study programme without an explicit endpoint is harder to sustain than one with a visible horizon. The current operational target is **M48 (~Jun 2030)** baseline + **M50 (~Aug 2030)** W13-trigger fallback + **M52 (~Oct 2030)** sustained-overage stretch fallback (per the 2026-05-25 M48 rebuild — see execution_plan.md §A.1 and the Capacity envelope report). Trimming strategies + falling-behind protocols (execution_plan §A) should keep this horizon in mind — *but* the target is not a constraint to be defended at the cost of capacity-sustainable pacing. The M48 anchor itself was set by accepting the previous M44 motivational anchor's collision with the scope × capacity arithmetic: rather than overcommit weekly hours or take Layer 2 cuts that would touch the community-engagement apparatus, the timeline was extended. The 48-month frame keeps motivation; the capacity envelope keeps the programme sustainable; if they conflict, sustainability wins.

**Weekly rhythm**
- ~1h/day, ~7.5h/week sustainable
- ~40 productive weeks/year (the other ~12 absorb annual leave, illness, family, ECB peaks — NOT additional capacity)
- → ~300h/year baseline study time

**Project-allocated leave**
- 7–8 leave days/year project-allocated (out of ECB total 30 days/year)
- One dedicated leave-week per year of concentrated writing/retreat work
- → ~140–160h across the programme from transition retreats R1–R5 + writing blocks W1–W2

**Other capacity**
- GovC afternoons (when EU regulations release the reader from board-prep duties): ~16h/year
- Community engagement track (Phases II–III only): ~25–30h/year, drawn from the same project-allocated leave + evenings/weekends

**Aggregate capacity**: ~346h/year sustained = ~28.8 h/month. Programme envelope: ~1,384h at M48 baseline (48 × ~28.8) / ~1,442h at M50 fallback / ~1,499h at M52 stretch fallback. Community engagement track (~25–30h/y in Phases II–III) is additive on top of these envelopes per the Class A structural commitment.

**Voluntary overage band**: ~8–9 h/week is sustainable for short stretches (e.g., Module 7 peak; Phase III writing surges). Anything beyond ~10 h/week for more than 2–3 weeks is a falling-behind signal (see §3.4).

**The capacity ceiling is a structural commitment** (Class A per §8) — sustainability is the binding constraint. If a calculation says the programme needs more hours than this envelope provides, the response is to cut scope or extend timeline, not to overcommit weekly hours.

**Overage tolerance — resolved at M48 rebuild (2026-05-25).** Programme total demand on the Inv-as-basis ground-up tally is **~1,506h** (post-L0+L1-bonus marking). Against the M48 envelope this is **+8.8%** overage; M50 fallback is +4.4%; M52 stretch is +0.5%. All three sit within the 13% tolerance band carried over from earlier audits. The previously-flagged "~20% overage pending user decision" (Class C row in §6) is **resolved** by the L0+L1 bonus framework — Layer 0 (bonus reads + 4.6 Adorno *ND* trim, ~10–15h) and Layer 1 (genuinely Low-interlocutor SHOULDs/COULDs: 8.11 Allen-Mendieta, 7.11 J. For an outsider-insider profile this is upside-down. The user's *scarce, asymmetric signal* (ECB economist who can credibly engage Frankfurt-tradition material) is what should be made visible. The rebalance: downgrade audience-side activities; foreground activities where the user's profile is the value-add.

## 3. Structural Design Requirements for the Execution Plan

### 3.1 Weekly Assignment Structure

Each week should specify:
- **Core assignment** (~7–8 hours): the primary reading task for that week, drawn from the core programme. Specify exact pages/chapters, not just "continue Module X." Where a text spans multiple weeks, provide sub-assignments at the chapter or section level.
- **Bonus reading** (~2–4 hours, optional): a reading that enriches the current core assignment but is not required. Sources: (a) Context Layer items timed to the relevant core module; (b) [RS] items from the current module that didn't make the core cut; (c) secondary literature that illuminates the current primary text; (d) Phase IV connection points marked in the syllabus. The bonus reading should be genuinely optional — skipping it should never create a gap in the core trajectory. Label these clearly as "if you have extra time this week."
- **Active-practice task** (where applicable per the tiered regime): what to write, how long it should be, what question it should answer.
- **Session structure hint**: a brief annotation indicating what kind of sessions the current text rewards. Not a rigid schedule — a suggestion the reader can adjust. See §3.5 for the text-type → session mapping. The default rhythm is ~1h/day on 5–6 days plus a longer block (~2h) on one day for writing tasks. Deviations from the default are flagged only where the text genuinely requires it.

### 3.2 Comprehension Infrastructure

**Text-level checks (new — not in the syllabus).** For each primary text that takes >10 hours, provide 2–3 specific comprehension questions the reader should be able to answer after completing it. These should test argumentative understanding, not recall. Format: "After reading X, you should be able to: (a) reconstruct the argument that...; (b) explain why the author claims...; (c) identify where the argument is vulnerable to the objection that..."

**Module-level checks (from the syllabus).** The syllabus already provides these ("Comprehension indicator: after this module..."). Reproduce them at the appropriate week in the execution plan.

**Deviation protocols.** Include guidance for when actual time deviates significantly from estimated time:
- "If you've spent >X hours and can't answer the text-level comprehension questions, switch to secondary text Y for 5–8 hours, then return."
- "If you finished in <X hours and the comprehension checks feel easy, you may be reading too fast — test yourself by attempting a written reconstruction before moving on."
- "If a text is defeating you and the secondary doesn't help, skip to the next assignment and flag it for return. Do not spend more than 150% of the estimated time on any single text."

### 3.7 Concentrated time blocks

The weekly rhythm (§2.1) is sustainable but inadequate for two kinds of work: (i) phase-spanning synthetic overviews that require holding many texts in mind at once; (ii) coherent long-form prose written without losing the argumentative arc between sessions. Concentrated blocks compensate. Two pools (per §8 Class A):

- **Pool 1**: project-allocated ~7–8 leave days/year → **5 transition retreats R1–R5** anchored to module/phase boundaries
- **Pool 2**: dedicated leave-week ~5 days × ~6h = ~30h/year → **2 writing blocks W1/W2** (M40 W3.2 first draft + M47 final revision, per M48 rebuild)

**Transition retreats** (~16–20h each, ~85h total). R1 (W8): post-Part-0 field overview. R2 (W33–34): Hegel–Marx synthesis. R3 (W63–64): Phase I synthesis + Honneth "Pathologien" re-read. R4 (M16 W1): Phase II opener + Vorlesungsverzeichnis check. R5 (M35 per M48 rebuild; was M30 pre-rebuild): Phase III opener + W3.2 candidate-topic outline + conditional-MUST (Layer C) promotion decisions (per [importance_tiers.md](importance_tiers.md) §4 and execution_plan §A.1 Layer C).

**Writing blocks** (~30h each, ~60h total). W1 at M40 per M48 rebuild (was M36; W3.2 first-draft consolidation; 5 days × 6h; pre-book to avoid GovC). W2 at M47 per M48 rebuild (was M43; final-revision consolidation; same constraints; avoid Christmas).

**Why concentrated blocks matter.** Daily 1-hour fragments cannot generate a 7,000-word philosophical argument with a single argumentative arc (W1/W2) or a phase-spanning synthetic overview (R1–R5). Retreats also function as enhanced checkpoints — the synthetic overview reveals misunderstandings that module-level comprehension checks might miss.

Operational details (per-retreat tasks, leave-booking lead times, GovC conflict avoidance) live in [execution_plan.md](execution_plan.md) Appendix E (retreats) + Appendix F (writing blocks).

## 5. Key Tensions the Execution Plan Must Navigate

**A. Early production vs. premature production.** Written output is deferred to the Motivational Checkpoint (M14, W57 Field Report) beyond Light-tier per-chapter Recon work. Craft Block production begins in Phase III. The Comparison Exercise (M37–M38 per M48 rebuild; was M32–34 pre-rebuild) is inserted between W3.1 and W3.2 as a bridge: harder than reconstruction, easier than original critique, and necessary for the positioning skill that the publishable piece requires.

**B. Comprehensiveness vs. feasibility.** Phase III's MUST-Anchor/MUST-Engage/SHOULD/COULD/DISCRETIONARY classification preserves intellectual breadth (via SHOULD + Context-Layer-style Orientation reads) without inflating the core budget. The execution plan keeps tier annotations visible in the Inventory so the reader always knows whether a given text is anchor-apparatus (Full), reviewer-coverage (Light), or breadth-context (Orientation).

**C. German vs. English.** The reader is fluent in German. Where both versions exist, German is generally preferred for Frankfurt-tradition primary texts (Hegel, Marx, Habermas, Honneth, Jaeggi, Adorno) because the conceptual vocabulary is canonical in German. English is fine for secondary literature, analytic philosophy texts, and non-German-tradition authors (Dewey, Fricker, Taylor, List & Pettit). Pacing in the Style × Tier matrix accounts for this: DE pp/h is slightly slower than EN within the same prose class.

**D. Community engagement vs. reading time.** The ~25–30 hours/year allocated to engagement in Phases II–III is a structural commitment per §8 Class A, not buffer-drawdown. Acceptable because (a) the comprehension benefit of live philosophical engagement likely saves more than 25 hours of self-correction per year, and (b) the community engagement directly serves the programme's ultimate goal (becoming a recognised interlocutor), which pure reading does not. Cut order under falling-behind: bonus readings → SHOULDs → tier downgrades → engagement reduction. **Never cut Vorlesung mid-semester unless behind 8+ weeks.**

**E. Slippage vs. re-engagement.** The dominant failure mode for multi-year self-study is not intellectual difficulty but schedule slippage leading to motivational collapse. The plan's falling-behind protocols ([execution_plan.md](execution_plan.md) §A + §A.1) are calibrated against this: graduated 2/4/8-week triggers, and at the 8-week threshold the rule is re-entry at the next phase boundary rather than catch-up. The goal is re-engagement, not checklist completion.

**F. Falling-behind protocol additions (2026-07-13, plan audit) — owned here; execution plan v3 implements both in its Appendix A.**

- **Dependency-stub rule.** When a falling-behind cut or the 8-week skip rule drops an item that [importance_tiers.md](importance_tiers.md) §4 (or the syllabus dependency graph) lists as a prerequisite of a later MUST-Anchor, the item is not dropped to zero: it is replaced by an **Orientation-tier dependency stub** — ~1–2h from secondary literature, aimed at knowing the argument's *shape*, not the text. Purpose: preserve the anti-collapse function of skip-don't-catch-up (§5.E) while protecting prerequisite chains.
- **Practice-debt trigger.** The falling-behind triggers currently watch only the reading schedule; a reader can be on schedule and have silently stopped writing. Addition: **two or more tier-mandated filed notes missing at a weekly review = practice debt**, which fires like the 2-week reading trigger. The sanctioned response is same-week **degradation-in-place** (Full reconstruction → Light note → reading-log line) — never silent omission, never catch-up writing. Canonical method definition: [reading_practices.md](reading_practices.md), "Practice debt and the degradation floor" (Part III).
