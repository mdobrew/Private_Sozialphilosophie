## 1. What "Phase III frontier" means

**Target outputs:**
- Phase III publication-cycle deliverables (per [bridge_document.md](bridge_document.md) §2.1 and the Phase III production cycle in [execution_plan_v1.md](execution_plan_v1.md) Months 27–42):
  - One 7,000–10,000-word article (C3.2 draft) submitted to a Frankfurt-tradition journal in M42
  - One informal working-group / IfS-style presentation (M35–37)
  - One Tagung/workshop presentation (M37–40)
  - One critical review (~1,500 words, C3.3)

**Target journals (ranked by relevance to Sozialphilosophie):**
- *Deutsche Zeitschrift für Philosophie* (DZfPh) — core German-language venue
- *WestEnd: Neue Zeitschrift für Sozialforschung* — Institut für Sozialforschung journal
- *Zeitschrift für kritische Sozialtheorie und Philosophie* (ZkSP) — dedicated to critical social theory
- *Constellations* — core Anglophone
- *Philosophy & Social Criticism* — high relevance
- *Critical Horizons* — recognition-theory focus
- *European Journal of Philosophy* — accepts Sozialphilosophie

**Reviewer norms presupposed:**
- Frankfurt-tradition reviewers expect engagement with current debates (Honneth–Fraser, Jaeggi, Forst, Allen) and with active frontier voices (Khurana, Vogelmann, von Redecker, Menke).
- Empirical grounding accepted but not required; methodological self-awareness about reconstruction vs. constructivism vs. genealogy is required.
- Bridging to capitalism critique (Streeck, Postone, Fraser, Polanyi) expected if the contribution touches political-economy themes; expected NOT to substitute for normative-philosophical engagement.
- Engagement with external challenges (postcolonial / decolonial / Black radical via 8A; standpoint epistemology via 9.8–9.10) is increasingly expected at frontier venues; absence is now reviewable.

**Phase III contribution shape (assumed default, revisable as topic firms up):**
- A single contribution-length piece in Sozialphilosophie, defensible against Frankfurt-tradition reviewer norms
- Likely terrain: social pathologies / recognition / immanent critique / capitalism-as-form / methodological reconstruction-vs-alternatives
- Open territory pending Phase III topic firm-up (Months 22–28, per [execution_plan_v1.md](execution_plan_v1.md))

**Classification is provisional.** Triggers for re-classification (see §5):
1. Phase III topic firms up around a specific subfield → conditional COULDs auto-promote to MUSTs in that subfield
2. Falling-behind 4-week+ protocol activates → SHOULDs are first to cut, then COULDs, then DISCRETIONARY (per [bridge_document.md](bridge_document.md) §3.4)
3. New frontier text appears that displaces an existing one
4. External feedback (workshop discussant, scholar correspondence) identifies a missing piece

## 2. Categories and tier-derivation rule

### 2.1 The four importance categories

A text is **MUST** if either it is reviewer-expected for the Phase III piece (Phase III-reviewer-essential) or it is tradition-anchoring (cannot engage Frankfurt social philosophy without it) — the criterion is the disjunction (a) ∨ (b).

| Category | Definition | Failing-behind treatment |
|---|---|---|
| **MUST** | (Phase III-reviewer-essential) ∨ (tradition-anchoring). Cannot be cut without compromising defensibility in front of Frankfurt-tradition reviewers. | Preserved. If cut, escalate to topic re-scope. |
| **SHOULD** | Supports Phase III directly but publication could proceed without. Strong recommendation; provides depth and competing-positions awareness. | First to cut under 4-week+ falling-behind. |
| **COULD** | Relevant *only if* Phase III topic commits to specific terrain. Conditional. Promote to MUST-Anchor / MUST-Engage / SHOULD if topic firms up around the relevant theme. | Defer to Phase IV if Phase III topic does not commit. |
| **DISCRETIONARY** | Of intellectual interest; not Phase III-load-bearing. | Excluded from active schedule. Available as Phase IV topical-deepening. |

### 2.2 MUST sub-split — Anchor vs Engage

MUST has two sub-classes, distinguished by *whether the user will reconstruct the text's apparatus in his own voice* (Anchor) or *know it well enough to cite/characterise* (Engage):

| Sub-class | Definition | Default tier | Test question |
|---|---|---|---|
| **MUST-Anchor** | Apparatus the user will deploy under his own pen in the Phase III piece. Apparatus-reuse. | **Full** | "Will I redeploy this text's conceptual machinery to make my own argument?" |
| **MUST-Engage** | Reviewer-expected coverage. Must read with comprehension, characterise, position-take, but no reconstruction-as-apprenticeship. | **Light** | "Will I cite and engage this without using it as my own scaffolding?" |

The MUST = (a) ∨ (b) criterion identifies membership; the Anchor/Engage split tracks *which* criterion drives. The intersection of (a) and (b) is typically Anchor; texts satisfying only one of the two criteria are typically Engage.

### 2.3 Tier-derivation rule — y3 class first, syllabus tag exceptions

| y3 class | Default tier | Tag-driven exceptions |
|---|---|---|
| **MUST-Anchor** | Full | (none — y3 class dominates regardless of tag) |
| **MUST-Engage** | Light | [O*]/[O]/[RS] → Orientation (read with reading-log line, no filed Recon) |
| **SHOULD** | Light if [P]/[ES]/[ES*]; Orientation if [O*]/[O]/[RS] | — |
| **COULD** | Orientation; if §4 trigger fires, adopt tier of the promoted class | — |
| **DISCRETIONARY** | Skip / Phase IV | — |

**Important — this rule supersedes the tag-only default mapping in execution_plan_v1.md Appendix A.** The execution plan's Inventory Tier-column should derive from `(y3 class × syllabus tag)`, with y3 class dominant. Where the syllabus tag would imply a different tier than y3 (e.g., [P*]-tagged Zurn 8.3 would be Full by tag but is MUST-Engage = Light here), y3 wins and the Inventory annotation `[P*] → Light (per y3 MUST-Engage)` makes the deviation explicit.

### 2.4 Asymmetries this captures

1. A MUST [P] can be either Full or Light depending on apparatus vs coverage. *Phänomenologie* IV.A (1.3) is MUST-Anchor [P] → Full because Herr/Knecht is the conceptual entry to recognition theory.
2. A MUST [ES] / [ES*] can be Full when it is the *portal* through which the user controls a primary apparatus he will reuse. Heinrich (2.3) is MUST-Anchor [ES] → Full; Neuhouser (1.1) is MUST-Anchor [ES*] → Full.
3. A MUST [P*] can be Light when the [P*] tag captures "best frontier statement" rather than "apparatus to reuse." Zurn 8.3, Laitinen-Särkelä 8.4, Jaeggi-Fraser 8.5 are MUST-Engage [P*] → Light because they are reviewer-expected coverage, not apparatus.
4. SHOULD [P*] / [ES*] should not exist; if it does, that is a flag to re-examine the y3 class, not a routing rule.
