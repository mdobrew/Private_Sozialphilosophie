## 2. Scored findings

Scoring 1–5 (5 = excellent). Where resolution strata exist, each stratum is scored *separately* (no averaging). Weakness type: **Structural** (reorg) / **Local** (fixable in place) / **Calibration-dependent** (needs empirical input; decision rule stated).

### D1 — Intellectual narrative / throughline — **5 (core modules) / 4 (Part III instrumental modules) / 3 (appendices)**

- **Indicators (good):** every module opens with a "Why this matters" that names the *forward* payload (e.g., Lukács 3A as the "missing step" Marx/Weber→Frankfurt; Mead 4A.3 placed so the reader "arrives at Honneth with the social-psychological foundations already in hand"); the [MX] cross-references thread a coherent secondary spine; "the question this module answers for the Sozialphilosophie student" closes most Part III modules.
- **Core stratum (5):** the central chain is genuinely argument-shaped, not list-shaped — exemplary.
- **Part III stratum (4):** the rival-paradigm framing (11A–11D as "each forces the reader to articulate commitments that might otherwise remain implicit") is strong, but Module 15 is explicitly "a menu, not a sequence," and Modules 16/17/18 read as orphaned reference shelves with thinner forward-connection than the core.
- **Appendix stratum (3):** A/B/F/G are competent bibliographies but the throughline weakens to "deepening lists"; defensible for reference material, scored for direction only.
- **Weakness type:** Local (the throughline is present; only the Part III tail and appendices under-deliver it).

### D2 — Dependency-graph & progression correctness — **4 (DAG) / 3 (prose cross-refs vs DAG)**

- **DAG stratum (4):** the Mermaid graph is acyclic, the "critical path" narration is coherent, soft vs hard edges are distinguished, and the convergence/free-variable analysis is genuinely useful. One internal inconsistency: the DAG node is labelled **"Appendix G — Socialisation"** (line ~1356) and the reading-the-graph note says "Appendix G (Socialisation)" (line ~1386, ~1402), but the actual section header is **"APPENDIX H: SOCIALISATION…"** — the socialisation appendix is H, not G (G is Social Phenomenology). Mislabel, not a cycle.
- **Prose-vs-DAG stratum (3):** several in-text [MX]/cross-references point to stale or wrong item numbers — e.g. Module 16's [MX] note cites "Kosík… (Module 13 item 13.12)" but Kosík is `C.12` (there is no 13.12); Module 4 and 3A [MX] notes point to "Appendix C item C.12" correctly elsewhere, so the pointer convention is inconsistently applied. These are navigability defects that ride on top of an otherwise-correct dependency structure.
- **Weakness type:** Local (fix labels/pointers in place; DAG topology itself is sound).

### D3 — Marker-system consistency — **4**

- **Indicators (good):** the marker key is explicit; [Economist's note] markers are preserved and substantive; [MX] and Phase-IV connection flags are consistent and load-bearing; the [P]→[RS] / [NE]→[ES] revision annotations (4.4, 4.11, 6.6, 6.7, 7.6, 9.6, 9.9, 10.4, 12.2) are dated and traceable.
- **Defect:** the accelerated-path asterisk system is under-maintained. The marker key promises asterisked items "are marked … in the module tables above," and `phase3_critical_path` v2.1 promoted **9.2 Fricker, 5.2 Habermas EuI, 7.7 Forst, 8.6 Allen, 3A.1 Lukács** to MUST-Anchor (Full) — i.e. they are now treated as structurally indispensable — yet the syllabus accelerated-path list and the in-table `[*]` tags were set pre-v2.1 (Fricker/Taylor appear on the path; Forst/Lukács/EuI/Allen do not). The asterisk set and the MUST-Anchor set have drifted apart. This is a *consistency* issue, not a call to change the marker system.
- **Minor:** module "core without [NE]" sub-totals occasionally equal the all-in total even where an [NE] item is listed (e.g. Module 2 "core without [NE]: 55–72" equals the all-in 55–72 although 2.4 Wood [NE] is in the table) — the parenthetical is internally inconsistent in a handful of modules.
- **Weakness type:** Local.

### D4 — Workload calibration (hour estimates) — **2 (per-text "Est. hours" column) / 3 (part/grand totals) / 4 (relative ordering)**

This is the syllabus's weakest structural dimension and overlaps heavily with D8.

- **Per-text stratum (2):** the "Est. hours" column is **not reconcilable with the canonical §C.2 Style × Tier matrix** that downstream docs treat as authoritative. Spot-checks at bottom-of-range pp/h, Full tier:
  - Hegel *RP* 1.2 (Dense DE, ~320pp, 4 ch): matrix → **~72h**; syllabus says **40–60h**.
  - Marx *Kapital* 2.2b (Dense DE, ~160pp): matrix → **~38h**; syllabus **22–30h**.
  - Neuhouser 1.1 (Transparent EN, ~300pp): matrix → **~19.5h**; syllabus **25–35h**.
  - Hegel *Phän* 1.3 (Dense DE, ~30pp): matrix → **~8h**; syllabus **10–15h**.
  - Freyenhagen 4.6, Fricker 9.2: matrix → **~13h**; syllabus **15–20h**.
  The pattern is systematic: the syllabus *under*-estimates Dense-DE primaries (where the matrix's 5 pp/h is genuinely slow) and *over*-estimates Transparent/Analytic secondaries. The two estimators disagree by 20–50% in both directions, so the syllabus column is not a noisy version of the matrix — it encodes a different (pre-matrix, tier-blind) pacing model.
- **Totals stratum (3):** part/grand totals are *internally* additive and the "all-in vs scheduled subset" decomposition (Part II ~430–586 all-in vs ~484 scheduled) is honest, but they inherit the per-text model and therefore diverge from any matrix-derived schedule. The "Summary of Time Estimates" grand total (~1,553–2,134 core) is a *different object* from the bridge envelope (~1,384h M48) and the phase3 demand tally (~1,506h); a first-time reader cannot tell that the syllabus number is "full-syllabus all-tier, not the committed programme."
- **Relative-ordering stratum (4):** as a *rank-ordering* of effort (which texts are big/small) the column is fine and useful for orientation.
- **Decision rule (Calibration-dependent):** do NOT hand-reconcile the column to the matrix text-by-text. The right move is a one-line *disclaimer* at the column's first use stating that "Est. hours" is an orientation-grade, tier-blind figure superseded for scheduling by the §C.2 matrix as instantiated in `execution_plan` Inventory. Whether the column should be *regenerated* from the matrix is a Class B change (matrix is Class B) and must be flagged for propagation — but the cheaper, scope-respecting fix is the disclaimer. The W9–W13 calibration log is the empirical input that would later tell you whether the matrix itself (not the syllabus column) needs its pp/h re-tuned.
- **Weakness type:** Calibration-dependent (per-text); Local (the disclaimer).

### D6 — Reference-usability / navigability — **2**

The single most fixable cluster of defects, and the one that most undercuts the document's stated purpose ("a reference document … strategic orientation … 'is it on the syllabus and where does it connect?'").

- **Appendix-letter collisions (Structural-lite):** the "Summary of Time Estimates" lists Appendices **A–H** with A=Social Epist, B=Phil Soc Sci, C=Marxist, D=Capitalism, E=Zeitdiagnose, F=Social Ontology, G=Phenomenology, H=Socialisation. But the appendix **bodies** carry mismatched total-lines: the Marxist appendix (header "APPENDIX C") prints "**Appendix D total**" (line ~736 region is actually App-A's "Appendix D total"; App-A header is "APPENDIX A" but closes "Appendix D total: 150–200"); the Social-Ontology appendix prints "**Appendix C total (additional beyond Module 14)**" twice (lines ~944/949); the Phenomenology appendix (header "APPENDIX G") closes "**Appendix F total**"; the Socialisation appendix (header "APPENDIX H") closes "**Appendix G total**" and its DAG node calls it "Appendix G." At least five appendix total-lines name a *different letter* than their own header. A reader cross-referencing "Appendix C item C.6 (Brenner)" from a Module-8 [MX] note has to reverse-engineer which physical appendix is meant.
- **Stale item pointers (Local):** "Module 13 item 13.12" (should be C.12); inconsistent "Appendix C item C.x" vs "Module 13 item 13.x" for the same Backhaus/Reichelt/Kosík items now that Module 13 was extracted from old Appendix C.
- **Verdict:** the *content* is navigable; the *labels* are not. Because the document's whole job is lookup, label integrity is disproportionately important here.
- **Weakness type:** Structural (the appendix-letter scheme needs one consistent pass) + Local (individual pointers).

- **Timeline / parameter spine (2) — the headline finding.** The syllabus is anchored to the **superseded M44 era**, not the current **M48 rebuild (bridge, 2026-05-25)**:

## 3. Recommended revision sequence (with dependencies)

Ordered so that each step de-risks the next. Steps 1–2 are propagation of *settled* upstream decisions (not re-openings); steps 3–5 are internal hygiene.

1. **Re-anchor the timeline spine to M48 — FIRST, and do it jointly with `phase3_critical_path.md`.**
   *Why first:* every downstream reference (R5 month, phase lengths, Phase-II windows, frontier-reading window, accelerated-path decision rule) inherits these coordinates. Doing it before any hour/total work avoids re-touching the same lines twice. *Dependency:* the syllabus and phase3 are *both* M44-stale; they should be re-anchored in the **same** pass (or phase3 first, since the syllabus's Part-II/accelerated-path prose cites phase3 §4). Treat as Class A propagation — copy the M48 spine from bridge §2.1/§6, do not re-derive.

2. **Decide the hour-column treatment (disclaimer vs regenerate) — SECOND, after the timeline is fixed.**
   *Why after 1:* part/module totals sit next to phase/month references; fix the anchors first. *Recommended (scope-respecting):* add a one-line disclaimer at the "Est. hours" column's first use + at the Summary-of-Time-Estimates header stating the column is orientation-grade and tier-blind, superseded for scheduling by §C.2/`execution_plan` Inventory. *If* the user wants the column regenerated from the matrix, that is a **Class B** change requiring propagation notes to `text_classification` and `execution_plan` — flag, don't silently fix.

3. **Single appendix-letter integrity pass (D6) — THIRD.**
   Reconcile every appendix *total-line*, *DAG node*, and *cross-reference pointer* to the A–H scheme in the Summary table. *Dependency:* independent of 1–2 but should follow them so the pass touches stable text. This is the highest navigability-ROI fix.

4. **Page-count anomaly reconciliation (D8 hour sub-axis) — FOURTH.**
   Adopt the `text_classification.md` "Notes on cross-listings and special cases" corrections (A.2.2, A.3.2, A.3.3, C.7, C.8, F.3.2, B.4.3) into the syllabus so the two stop disagreeing. *Dependency:* after 2, since it touches the same hour/page material.

5. **Marker/pointer micro-pass (D3/D2) — LAST.**
   Re-sync the accelerated-path asterisk set to the v2.1 MUST-Anchor set; fix the "13.12"→"C.12"-type stale item pointers; reconcile the handful of "core without [NE]" sub-totals that equal the all-in total. Lowest-risk, highest-tedium; do once everything above is stable.

## 4. Most consequential design trade-offs already made

For each: the apparent choice, whether it's defensible given the goal (publishable Phase III contribution at ~7–8 h/week), and the evidence that would flip it.

1. **Two parallel hour estimators (syllabus column vs §C.2 matrix).**
   *Choice:* keep the narrative "Est. hours" column in the syllabus while making the matrix the canonical scheduler. *Defensible?* Partly — a reference map *should* carry quick orientation numbers, and forcing the matrix into a bibliography would clutter it. But leaving the two un-flagged and 20–50% divergent is not defensible; it invites mis-planning. *Flips if:* the W9–W13 calibration shows the matrix pp/h are themselves off (then regenerate the column from a corrected matrix); or the user finds the column is never consulted (then delete it rather than disclaim it).

2. **Comprehensiveness over selectivity at the document level (massive all-in syllabus, then prune via phase3/execution_plan).**
   *Choice:* the syllabus inventories ~core 1,553–2,134h + ~1,185–1,680h appendices (grand ~2,738–3,814h) and lets the *other* documents do the cutting. *Defensible?* Yes, given the stated role ("comprehensive map … not week-by-week execution") and the Class-A discipline that cuts live in phase3/execution_plan. The risk it accepts: a reader who consults only the syllabus will badly over-estimate the committed programme. The all-in/scheduled-subset annotations mitigate this in Part II but not in Parts I/III. *Flips if:* motivational evidence shows the sheer size is demoralising rather than orienting — then a visible "committed core ≈ X h" banner is warranted.

3. **Keeping the syllabus deliberately "modify rarely; structural only" while companion docs iterate fast.**
   *Choice:* let the syllabus lag (it is M44-anchored two rebuilds after the spine moved to M48). *Defensible?* As a stability anchor, mostly — but the lag has now reached the *timeline spine and retreat months*, which are Class A and load-bearing for navigation. There is a difference between "don't churn the bibliography" and "let the phase lengths go stale." *Flips if:* (already flipped) the staleness touches Class A coordinates — at that point propagation is mandatory, not optional.

4. **Asterisk accelerated-path as a *separate* tagging layer from phase3 MUST-Anchor.**
   *Choice:* two indispensability systems (syllabus [*] and phase3 MUST-Anchor) instead of one. *Defensible?* The two answer different questions ([*] = fastest route to *frontier-reading*; MUST-Anchor = apparatus the Phase-III *piece* will reuse), so divergence is partly principled. But after v2.1 promoted Lukács/EuI/Forst/Allen to Anchor, the *unflagged* divergence reads like an error rather than a distinction. *Flips if:* the user decides the two should be merged — then the [*] system should be regenerated from MUST-Anchor + accelerated-reading-order, a Class B change.

### Unintended tensions (improving one dimension degrades another)

- **D6 navigability fix vs D-stability mandate.** A thorough appendix-letter renumber maximises navigability but is exactly the kind of structural churn the "modify rarely" posture resists. *Resolution:* fix *labels/pointers to match the existing A–H scheme* (no renumbering of items), which buys navigability without re-IDing anything readers may have cited.
- **D4 hour-column accuracy vs D1 readability.** Regenerating per-text hours from the matrix would make D4/D8 consistent but injects tier-conditional numbers into a narrative map, hurting D1's orientation function. *Resolution:* the disclaimer route (don't regenerate) keeps D1 intact at the cost of leaving D4 formally "orientation-grade" — an acceptable trade for a reference document.
- **D5 frontier currency vs D2/D8 stability.** Each newly added 2023–2024 frontier text (good for D5) is another node the DAG, [MX] pointers, phase3 classification, and text_classification must absorb (load on D2/D6/D8). The syllabus has so far absorbed these well, but the rate of frontier additions is the main ongoing threat to cross-doc coherence. *Resolution:* the existing "cross-document update protocol" in phase3 §6 is the right control; ensure new syllabus additions trigger it.
