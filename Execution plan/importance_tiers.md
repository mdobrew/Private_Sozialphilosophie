## 2. Categories and tier-derivation rule

### 2.1 The four importance categories

A text is **MUST** if either it is reviewer-expected for the Phase III piece (Phase III-reviewer-essential) or it is tradition-anchoring (cannot engage Frankfurt social philosophy without it) — the criterion is the disjunction (a) ∨ (b).

| Category | Definition | Falling-behind treatment |
|---|---|---|
| **MUST** | (Phase III-reviewer-essential) ∨ (tradition-anchoring). Cannot be cut without compromising defensibility in front of Frankfurt-tradition reviewers. | Preserved. If cut, escalate to topic re-scope. |
| **SHOULD** | Supports Phase III directly but publication could proceed without. Strong recommendation; provides depth and competing-positions awareness. | First to cut under 4-week+ falling-behind. |
| **COULD** | Relevant *only if* Phase III topic commits to specific terrain. Conditional. Promote to MUST-Anchor / MUST-Engage / SHOULD if topic firms up around the relevant theme. | Defer to Phase IV if Phase III topic does not commit. |
| **DISCRETIONARY** | Of intellectual interest; not Phase III-load-bearing. | Excluded from active schedule. Available as Phase IV topical-deepening. |

### 2.2 MUST sub-split — Anchor vs Engage

MUST has two sub-classes, distinguished by *whether the user will reconstruct the text's apparatus in his own voice* (Anchor) or *know it well enough to cite/characterise* (Engage):

| Sub-class | Definition | Default tier | Test question |
|---|---|---|---|
| **MUST-Anchor** | Apparatus the user will deploy under his own pen in the Phase III piece. Apparatus-reuse. | **Full** | "Will I redeploy this text's conceptual machinery to make my own argument?" |
| **MUST-Engage** | Reviewer-expected coverage. Must read with comprehension, characterise, position-take, but no reconstruction-as-apprenticeship. | **Light** | "Will I cite and engage this without using it as my own scaffolding?" |

The MUST = (a) ∨ (b) criterion identifies membership; the Anchor/Engage split tracks *which* criterion drives. The intersection of (a) and (b) is typically Anchor; texts satisfying only one of the two criteria are typically Engage.

### 2.3 Tier-derivation rule — Phase III class first, syllabus tag exceptions

| Phase III class | Default tier | Tag-driven exceptions |
|---|---|---|
| **MUST-Anchor** | Full | (none — Phase III class dominates regardless of tag) |
| **MUST-Engage** | Light | [O*]/[O]/[RS] → Orientation (read with reading-log line, no filed Recon) |
| **SHOULD** | Light if [P]/[ES]/[ES*]; Orientation if [O*]/[O]/[RS] | — |
| **COULD** | Orientation; if §4 trigger fires, adopt tier of the promoted class | — |
| **DISCRETIONARY** | Skip / Phase IV | — |

**Important — this rule supersedes the tag-only default mapping in execution_plan.md Appendix A.** The execution plan's Inventory Tier-column should derive from `(Phase III class × syllabus tag)`, with Phase III class dominant. Where the syllabus tag would imply a different tier than Phase III (e.g., [P*]-tagged Zurn 8.3 would be Full by tag but is MUST-Engage = Light here), Phase III wins and the Inventory annotation `[P*] → Light (per Phase III MUST-Engage)` makes the deviation explicit.

### 2.4 Asymmetries this captures

1. A MUST [P] can be either Full or Light depending on apparatus vs coverage. *Phänomenologie* IV.A (1.3) is MUST-Anchor [P] → Full because Herr/Knecht is the conceptual entry to recognition theory.
2. A MUST [ES] / [ES*] can be Full when it is the *portal* through which the user controls a primary apparatus he will reuse. Heinrich (2.4) is MUST-Anchor [ES] → Full; Neuhouser (1.1) is MUST-Anchor [ES*] → Full.
3. A MUST [P*] can be Light when the [P*] tag captures "best frontier statement" rather than "apparatus to reuse." Zurn 8.3, Laitinen-Särkelä 8.4, Jaeggi-Fraser 8.5 are MUST-Engage [P*] → Light because they are reviewer-expected coverage, not apparatus.
4. SHOULD [P*] / [ES*] should not exist; if it does, that is a flag to re-examine the Phase III class, not a routing rule.
